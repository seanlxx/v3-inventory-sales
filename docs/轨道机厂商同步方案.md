设置页提供“预览同步”和“确认同步”。同步范围可选库存/售价/成本、销售；日期区间最多 31 天。

## 数据规则

- 只读同步，不回写盛码库存、售价、成本或机器配置。
- 商品按 `machine_id='轨道机'` + 归一化名称匹配；匹配不到时创建商品并记录映射。
- 多货道库存求和；售价取出现次数最多的价格，并列取最低；成本按库存加权平均。
- 销售单以 `(source='shengma', external_id)` 去重。
- 库存校准写 `stock_movements` 的 `adjustment`，不直接覆盖余额。
- 同步商品、售价、成本后再导入销售，最后校准库存。

## 凭据

生产环境使用 Cloudflare Secrets：

```powershell
npx wrangler pages secret put SHENGMA_USERNAME
npx wrangler pages secret put SHENGMA_PASSWORD
```

可选变量：`SHENGMA_BASE_URL`。本地开发通过 `.dev.vars` 注入，禁止把账号、密码、Cookie 写入源码、Markdown 或 Git。

## 风险与处理

| 风险 | 处理 |
| --- | --- |
| 盛码页面结构变化 | 解析失败时返回明确错误，不静默猜测导入 |
| 验证码 / 风控 | 提示用户到盛码后台处理后重试 |
| 高频登录 | 复用 Cookie 缓存，同一时刻只允许一个同步运行 |
| 商品误匹配 | 保留 `external_product_mappings`，必要时人工修正 |
| 重复导入 | 唯一索引和幂等写入兜底 |

## 回归
