## 3. Excel 来源（1号机 / 2号机）

- **导出位置：** zn 平台后台 → 账户信息 → 订单明细 → 导出 Excel
- **样例文件：** `订单明细_2026-05-01_2026-05-25.xlsx`（项目根目录，已被 `.gitignore` 排除，含手机号等 PII，禁止提交）
- **关键字段：**

| 列名 | 说明 | 入库映射 |
| --- | --- | --- |
| 订单号 | `visionpay…` 唯一订单号 | `sales_orders.external_id` |
| 状态 | `已完成` / `取消` | 仅导入 `已完成` |
| 设备编号 | `TBN5…` | 通过本表映射到 `machine_id` |
| 创建时间 | `YYYY-MM-DD HH:mm:ss` | `sales_orders.record_date`（取日期） |
| 商品条码 | `13位GTIN-货道号` | `products.external_id` 取 13 位前缀做匹配 |
| 商品名称 | 商品文字名 | 兜底匹配 |
| 商品单价 | 元 | `sales_items.unit_price_cents` |
| 商品数量 | 整数 | `sales_items.quantity` |
| 销售额 | 元 | `sales_orders.total_amount_cents` |
| 退款金额 | 元 | 大于 0 的当前一律跳过（按需后续处理） |

- **多商品订单：** Excel 中同一 `订单号` 出现多行视为同一销售单的多个明细。
- **去重键：** `(integration='zn', external_id=订单号)`，重复导入会跳过。

## 4. 3号机（盛码）抓取

- 配置详情见 `docs/三号机厂商同步方案.md`。
- 入库流程与 zn Excel **共用** `_shared/integrations/sale-importer`，仅 client / 解析层不同。
- 触发方式：设置页内"预览同步 / 确认同步"按钮。

## 5. 命名约定

- 在代码、UI、报表中**一律使用**「1号机 / 2号机 / 3号机」作为 `machine_id`。
- 厂商显示名（如"工厂测试47T5D3"）只在导入摘要 / 调试日志中保留，不进业务库。
- 新接入设备时，先在本表登记一行，再在代码里加映射，禁止在散落文件里硬编码新编号。
